# NEXT STEP

> **Decide What to Learn Next**

학습 데이터를 '해석'이 아닌 '다음 행동'으로 연결하는 UX 중심 학습 분석 플랫폼

---

## 🔍 프로젝트 개요

**NEXT STEP**은 온라인 학습 플랫폼에서 발생하는 학습 데이터를
단순히 시각화하는 데 그치지 않고,
운영자와 학습자가 **'지금 무엇을 해야 할지' 즉시 결정할 수 있도록** 돕는
**UX 중심 Learning Analytics 대시보드**입니다.

기존 학습 분석 서비스는 이탈률·완강률 등 수치를 제공하지만,
사용자가 이를 해석하고 다음 행동으로 옮기기까지는 여전히 많은 부담이 존재합니다.

NEXT STEP은 이 문제를 UX 관점에서 재정의하여
**데이터 → 인사이트 → 행동**으로 자연스럽게 이어지는 흐름을 설계했습니다.

---

## 🎯 UX 문제 정의

### 기존 학습 분석 UX의 한계

- 데이터는 많지만 **어디서 문제가 발생하는지** 한눈에 보기 어려움
- 이탈률은 존재하지만 **우선적으로 개입해야 할 구간**이 드러나지 않음
- 학습자는 **자신의 수준과 다음 학습 경로**를 스스로 판단하기 어려움

### ✅ UX 목표

| 목표 | 설명 |
|------|------|
| **즉각적 인지** | 사용자가 10초 안에 가장 중요한 문제를 인지 |
| **행동 연결** | 분석 결과가 구체적인 행동 제안으로 연결 |
| **자기 이해** | 학습자가 자신의 상태를 이해하고 다음 학습을 선택 |

> UX를 디자인 문제가 아닌 **정보 전달과 의사결정 문제**로 접근했습니다.

## 🧩 핵심 UX 기능

### 1️⃣ Dropout Analysis — 학습 이탈 분석

**UX 관점**
- 단순 이탈률 나열 ❌
- **위험 구간 중심 시각화**로 즉각적인 우선순위 판단 가능 ✅

**UX 효과**
- 운영자가 "지금 개입해야 할 구간"을 바로 인지
- 데이터 해석 부담 감소

---

### 2️⃣ Course Analysis — 강의 상세 분석

**UX 관점**
- 개별 지표를 오가며 비교 ❌
- **퍼널 구조**로 학습 흐름을 한 번에 이해 ✅

**UX 효과**
- 수치를 분석하지 않아도 학습 흐름의 **병목 지점**을 직관적으로 파악
- 비율에 따른 **색상 피드백** (초록/노랑/빨강)으로 상태 즉시 인지

---

### 3️⃣ Level Test & Recommendation — 다음 행동 제안

**UX 관점**
- 점수 중심 평가 ❌
- **학습 단계 기반 결과 + 다음 학습 경로 제안** ✅

**UX 효과**
- 학습자가 자신의 현재 위치를 이해
- "다음에 무엇을 해야 할지" 명확해짐

## 🧭 상태 UX 설계 (Loading / Error)

### 문제
- 로딩·에러 상황에서 사용자가 기다리는 이유를 알 수 없음
- 에러 발생 시 다음 행동이 없음

### 해결
- **Loading**: 명확한 상태 메시지 제공
- **Error**: 재시도 가능한 액션 제공

> **UX 원칙** → 시스템 상태는 항상 사용자에게 명확히 전달

---

## 🏗️ UX를 고려한 구조 설계

### 공통 컴포넌트 설계
- `Button` / `Card` / `Input` / `Loading` / `ErrorMessage` 등 공통 UI 컴포넌트 분리
- 화면 간 **UX 일관성** 유지
- 사용자가 화면마다 다른 규칙을 학습하지 않도록 설계

### useApi 커스텀 훅
- API 호출 상태(`loading` / `error` / `data`) 통합 관리
- 모든 페이지에서 **동일한 상태 UX** 제공

> 구조 설계 역시 **사용자 경험의 일부**라고 판단했습니다.
